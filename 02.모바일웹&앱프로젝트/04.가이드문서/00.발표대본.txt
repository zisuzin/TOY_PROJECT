1. 비디오 슬라이드:
- 비디오를 리스트 3개로 묶어서 슬라이드로 만들었습니다.


    //아래 프로그래스바는 각 비디오마다 영상 최대길이와 인라인요소의 최대 퍼센티지를 비례시켜서, 영상이 끝나면 인라인요소 크기가 100%로 꽉 차도록했습니다.

    //  실시간으로 영상 속도에 맞게 바가 움직이도록 타임업데이트 이벤트도 사용했습니다. 영상의 현재 진행상태를 집계시켰는데, 타임업데이트의 이벤트 속도 자체가 느려서 모션이 버벅거리는 이슈가 있었습니다. 그래서 그부분은 setInterval로 보완해서 영상 진행상태를 빠른속도로 집계시키고 부드러운 모션이 나오도록 했습니다.

[나는 오늘 상큼 발랄걸이다 완전 가뿐한 도시여자 스타일로 발표할거임 - 대학시절 서혜미 빙의해]

홈페이지 대문에는 영상을 사용했구요, 아래에는 영상의 현재 상태를 보여주는 프로그래스바도 제작했습니다.
1차에서 진행했던 거에서 좀더 발전시켜서, 비디오를 슬라이드 형식으로 묶어봤구요,
영상이 종료되면 다음 영상을 호출하고 재생시키도록 코드를 짜봤습니다.

밑에 인기제품란은 제품을 다중으로 노출하게 슬라이드를 사용했습니다.
(요 부분은 슬라이드로 구성됐구요, 빠르게 넘기겠습니다~~)

그리고 제품 카테고리 란이구요, 클릭하면 해당 카테고리로 바로 이동하게 되어있습니다.

여기는 다량의 데이터 쭉 나열한거구요,

이 부분은 국내 오프라인 스토어 전경을 쭉 보여주는 파트입니다.
클릭하면 배경이미지가 클릭된 이미지 데이터로 바뀝니다!

아래부터는 하드코딩이라서, 넘어가겠습니다.

((네비게이션!))
이제 네비게이션 보시면, 여기부터 VUE.JS를 사용했습니다.
대분류를 총 네가지로 해서 각각의 데이터에 담았구요
이제 클릭했을때 펼쳐지는 아래 영역이 공통으로 사용되는 구역입니다.
대분류1번을 클릭하면 펼쳐지면서 해당 데이터가 셋업되고,
대분류2번을 클릭하면 해당 데이터의 리스트들이 또 셋업되는 방식입니다.

여기서 중분류, 소분류 클릭시 서브페이지로 넘어가구요, (:클릭한다)
((서브페이지로~~))
서브페이지 보시면, LNB가 따로 있습니다.
일단, 제 홈페이지가 index와 서브페이지가 따로 있기 때문에 그부분을 get방식을 통해서 연결시켜 줬습니다.
클릭된 gnb의 문자데이터를 키값으로 보내서 서브페이지에서 받아와서 쓰도록 했습니다.
이동된 페이지에서 받아온 데이터를 분석시킨다음 매칭되는 lnb 부분에 클릭 이벤트가 발생하도록 강제 트리거를 줬습니다.
그러면 해당 페이지 내용이 보시는 바와 같이 문제없이 들어옵니다.

그리고 lnb는 전체보기에서 하나씩 걸러지는 필터 형식으로 구성했구요,
v-if문을 사용해서 각 메뉴 순번과 데이터에 셋업된 고유 넘버 0,1,2,3을 비교시켜서 클릭시 매칭되는 데이터끼리만 걸러지도록 코딩했습니다.
